Project Checklist

The Project Checklist module enables users to configure and manage checklists for project tasks, ensuring tasks progress only when assigned checklists are completed. It includes stage-wise validation, user-specific assignments, progress tracking, and automated email reminders for pending checklists before the deadline

Highlighted-features

Checklist Assignment

Allows assigning checklists to specific users, stages, or projects, ensuring precise task tracking and responsibility delegation.

Progress Bar Tracking

Tracks checklist completion visually, updating the progress bar to reflect the status of task completion in real time.

Deadline Email Notifications

Sends automated reminders to users via email one day before the checklist deadline, keeping tasks on schedule.

Stage Transition Validation

Ensures that a task cannot move to the next stage until all associated checklists are completed.

Prevents skipping important steps by locking the stage transition process.

Promotes a structured workflow where each task is thoroughly checked before moving forward.

Checklist Completion Enforcement

Only the user to whom the checklist is assigned can mark it as completed, ensuring accountability.

Prevents unauthorized users or project managers from marking tasks as done, maintaining integrity.

Ensures that each checklist is completed by the designated person, improving task ownership.

Deadline Compliance with Auto Reminders

Sends email notifications to users one day before the checklist deadline to remind them of pending tasks.

Email notifications include task details and a direct link to the checklist, improving responsiveness.

Reduces the risk of missed deadlines by keeping users informed and on track for timely completion.

Checklist Configuration

  • Navigate to the Settings > Users & Companies > Click on Users > Go to Technical Sections > Enable the Manage Checklists in Project Task.

Checklists

  • Navigate to the Project > Go to Configuration menu > Select Checklist.

Creating Checklists

  • Now, create checklists, write checklist name, select projects ,stages , Assigned Users , Deadline

Create Multiple Checklist

Project-wise , Stage-wise and User wise assigning of Checklist

  • You can assign these checklists to any stage , user and project you want. Stages are listed according to the Project.

Checklist Visibility

  • Only visible checklist when match all record form checklist

No stage transition without completion of checklist at present stage

  • Try to move to the next stage, i.e., Task to IN PROGRESS, It will raise an error that all checklists are not done

Track Checklist Progress

Automatic Updation of Checklist

  • Checklist automatically gets updated after transition to new stage.

User wise Checklist Access

  • Only the assigned user can complete their checklist; if another user and project manager  attempts to complete it, a validation error will appear

reminder mail for task checklist

  • If the assigned user does not complete the checklist by the deadline date, send an email reminder one day before the checklist deadline.
